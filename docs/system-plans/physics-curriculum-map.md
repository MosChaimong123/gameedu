# Physics Curriculum Map

เอกสารนี้เป็น Phase 1 ของระบบ `Physics Lesson Template Master` ใช้สำหรับวางโครงแผนที่หลักสูตรฟิสิกส์ก่อนออกแบบ template บทเรียน, AI prompt, lesson builder และ course pack บนแพลตฟอร์ม

## Scope

- วิชา: ฟิสิกส์
- กลุ่มสาระ: วิทยาศาสตร์และเทคโนโลยี
- ประเภทรายวิชา: รายวิชาเพิ่มเติม
- ระดับ: มัธยมศึกษาตอนปลาย
- หลักสูตรอ้างอิง: หลักสูตรแกนกลางการศึกษาขั้นพื้นฐาน พ.ศ. 2551 ฉบับปรับปรุง พ.ศ. 2560

## Source Priority

ลำดับความสำคัญของแหล่งอ้างอิงที่ใช้กำหนดแผน:

1. สสวท. ใช้เป็นแหล่งอ้างอิงหลักด้าน curriculum backbone, learning outcomes และกรอบสาระฟิสิกส์
2. Aksorn ใช้เป็น reference ด้านการจัดหมวดระดับชั้นและการออกแบบชุดเนื้อหา
3. MacEducation ใช้เป็น reference ด้านการจัดแพ็กหนังสือ/หมวดเรียนระดับมัธยมปลาย
4. Platform Sequence ใช้เป็นลำดับการสอนเริ่มต้นของ TeachPlayEdu
5. Teacher Override ใช้สำหรับกรณีโรงเรียนต้องการสลับลำดับการสอนในภายหลัง

## Source-Backed Facts

ข้อมูลที่ยึดจากแหล่งอ้างอิงหลัก:

- สสวท. ระบุว่ากลุ่มสาระวิทยาศาสตร์และเทคโนโลยี ฉบับปรับปรุง พ.ศ. 2560 มีสาระเพิ่มเติมรวมถึงฟิสิกส์
- สสวท. ระบุว่าสาระฟิสิกส์มี 67 ผลการเรียนรู้ และจัดเป็น 4 หมวดใหญ่
- สสวท. เปิดช่องให้ครูหรือสถานศึกษาปรับลำดับการสอนได้ตามบริบท ตราบที่ยังครอบคลุมผลการเรียนรู้หลัก
- Aksorn และ MacEducation สะท้อนการใช้งานจริงในบริบทมัธยมปลาย และช่วยเป็น reference ฝั่ง product packaging

## Physics Content Groups From IPST

หมวดสาระฟิสิกส์ที่ใช้เป็น curriculum backbone:

1. กลศาสตร์
   ธรรมชาติทางฟิสิกส์ การวัด การเคลื่อนที่ แรง กฎการเคลื่อนที่ สมดุล งาน พลังงาน โมเมนตัม และการเคลื่อนที่แนวโค้ง
2. คลื่น แสง และเสียง
   การเคลื่อนที่แบบฮาร์มอนิกอย่างง่าย คลื่น เสียง แสง และปรากฏการณ์ที่เกี่ยวข้อง
3. ไฟฟ้า แม่เหล็ก และคลื่นแม่เหล็กไฟฟ้า
   ไฟฟ้าสถิต สนามไฟฟ้า วงจรไฟฟ้า แม่เหล็ก การเหนี่ยวนำ และคลื่นแม่เหล็กไฟฟ้า
4. ความร้อน ของไหล และฟิสิกส์สมัยใหม่
   ความร้อน แก๊ส ของแข็ง ของไหล ฟิสิกส์อะตอม ฟิสิกส์นิวเคลียร์ และฟิสิกส์อนุภาค

## Mapping Rule

เอกสารนี้แยกให้ชัดระหว่าง:

- `Curriculum Backbone`
  คือกรอบสาระที่ยึดตาม สสวท.
- `Platform Recommended Sequence`
  คือลำดับการจัดบทเรียนเริ่มต้นของ TeachPlayEdu

ถ้าโรงเรียนใดมีการสลับลำดับเรียนจริง ระบบควรอนุญาตให้ครู override ได้ แต่ห้ามทำให้ coverage ของสาระหลักหายไป

## Platform Recommended Sequence

ลำดับนี้ถูกปรับให้สอดคล้องตามภาพอ้างอิงที่ผู้ใช้กำหนด:

- ม.4 เทอม 1: ธรรมชาติและพัฒนาการทางฟิสิกส์, การเคลื่อนที่แนวตรง, แรงและกฎการเคลื่อนที่
- ม.4 เทอม 2: สมดุลกล, งานและพลังงาน, โมเมนตัมและการชน, การเคลื่อนที่แนวโค้ง
- ม.5 เทอม 1: การเคลื่อนที่แบบฮาร์มอนิกอย่างง่าย, คลื่น, แสงเชิงคลื่น, แสงเชิงรังสี
- ม.5 เทอม 2: เสียง, ไฟฟ้าสถิต, ไฟฟ้ากระแส
- ม.6 เทอม 1: แม่เหล็กและไฟฟ้า, ความร้อนและแก๊ส, ของแข็งและของไหล
- ม.6 เทอม 2: คลื่นแม่เหล็กไฟฟ้า, ฟิสิกส์อะตอม, ฟิสิกส์นิวเคลียร์และฟิสิกส์อนุภาค

หมายเหตุ: ลำดับนี้เป็น `platform default` เพื่อทำ template pack ให้เข้าใจง่ายและสื่อสารกับครูได้ตรง แต่ยังต้องยึด curriculum backbone ของ สสวท. เป็นแกน

## Curriculum Mapping Contract

ทุกบทเรียนฟิสิกส์ที่สร้างในระบบต้องผูกกับข้อมูลต่อไปนี้:

- `subject = physics`
- `curriculumCode = basic_education_2551_revised_2560`
- `gradeLevel`
- `semester`
- `unitId`
- `sourceGroup`
- `learningOutcomeIds`
- `sourceRefs`

ถ้าขาด field สำคัญข้อใดข้อหนึ่ง ห้ามถือว่าเป็น canonical physics lesson template

## Template Builder Notes

กติกาสำหรับเฟสถัดไป:

- 1 template ควรอิง 1 unit เป็นหลัก
- 1 unit สามารถมีหลาย lesson ได้
- 1 lesson สามารถผูกหลาย learning outcomes ได้ แต่ต้องมี `primary learning outcome`
- lesson title ควรเป็นภาษาที่ครูใช้จริง ไม่ใช่รหัส abstract อย่างเดียว
- ระบบควรให้ครูเพิ่ม school-specific emphasis ได้ โดยไม่ทำลาย curriculum backbone

## Teacher Override Rule

ครูสามารถปรับได้:

- ลำดับบทเรียน
- จำนวนคาบ
- ตัวอย่างโจทย์
- สื่อประกอบ
- กิจกรรมในชั้นเรียน

ครูไม่ควรแก้:

- subject หลัก
- curriculum code
- source reference หลัก
- การลบหน่วยหลักจน coverage ของสาระไม่ครบ

## Open Questions

ประเด็นสำหรับ Phase ถัดไป:

- จะเก็บ `learningOutcomeIds` เป็นรหัสจริงจากเอกสารหรือใช้ internal ids ก่อน
- จะมี template แยกตามสำนักพิมพ์ด้วยหรือใช้ canonical pack กลางก่อน
- จะมี `school pacing profile` สำหรับโรงเรียนที่สลับเทอม/สลับบทหรือไม่
- จะมี `simulation-ready lessons` สำหรับบทที่เชื่อม interactive หรือไม่
